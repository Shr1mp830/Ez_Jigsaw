# Ez_Jigsaw v1.0

一个纯本地 Python 桌面工具，用于处理 CTF 中常见的图片碎片拼图题。

## 功能

- 在界面中选择是否使用完整原图，不使用额外弹窗询问。
- 有原图时，将原图作为底图显示，并按原图区域相似度自动放置碎片。
- 无原图时，按碎片上下左右边缘像素差异打分，自动推断网格并拼接。
- 可输入一张已经乱拼成整图的图片，自动检测边缘并拆分成碎片。
- 可在 UI 中右键原图或碎片，选择 LSB 隐写通道与 bit0-bit7，结果直接显示在工作区画布。
- 可一键清除所有已加载图片，并自动废弃未完成的后台拆分/拼接结果。
- 大量图片导入后采用自适应缩放和分批渲染，降低 UI 卡顿和 Tkinter 崩溃概率。
- 支持图片边框自动对齐校准，自动拼接后按矩形网格收齐，手动摆放时按当前位置吸附。
- 兼容矩形碎片拼图，可手动填写行数和列数；填 `0` 表示自动推断。
- 拼接结束后停止，不会持续移动碎片。
- 支持手动拖动碎片微调。
- 支持选中碎片后左转、右转。
- 支持导出当前拼接结果为 PNG。
- 支持将拆分或当前加载的所有碎片导出到指定文件夹。
- 深色桌面 UI，左侧控制面板可滚动，顶部提供问题反馈入口。

## 运行

先安装依赖：

```powershell
python -m pip install -r requirements.txt
```

然后运行：

```powershell
python -m ctf_puzzle_solver
```

也可以直接双击 `run.bat`。

## 使用流程
[![屏幕截图 2026 06 07 135412](https://origin.picgo.net/2026/06/07/-2026-06-07-1354123a8f6fdadadb942c.png)](https://www.picgo.net/image/%E5%B1%8F%E5%B9%95%E6%88%AA%E5%9B%BE-2026-06-07-135412.1glHN4)




1. 启动后在左侧面板选择碎片文件夹，或选择“乱拼整图”后点击“拆分整图为碎片”。
2. 如果有完整原图，勾选“使用完整原图”并选择原图文件。
3. 如果知道矩形拼图大小，填写行数和列数；不知道就保持 `0`。
4. 拆分完成后不会自动拼图；可点击“加载项目”读取碎片，或先点击“导出所有碎片到文件夹”保存碎片。
5. 如结果不完美，直接在画布上拖动碎片修正。
6. 如碎片边缘有轻微错位，点击“边框对齐校准”自动收齐矩形边框。
7. 右键原图或碎片可选择 LSB 色道和 bit0-bit7，或在左侧 LSB 面板切换；显示会直接更新在工作区。
8. 点击“清除所有图片”可重置画布；点击“导出所有碎片到文件夹”保存碎片，点击“导出 PNG”保存拼接结果。

## 命令行自动拼接

无原图：

```powershell
python -m ctf_puzzle_solver.cli C:\path\to\pieces -o result.png
```

有原图：

```powershell
python -m ctf_puzzle_solver.cli C:\path\to\pieces -r C:\path\to\full.png -o result.png
```

指定矩形网格：

```powershell
python -m ctf_puzzle_solver.cli C:\path\to\pieces --rows 4 --cols 7 -o result.png
```




## GUI 命令行启动

```powershell
python -m ctf_puzzle_solver --pieces C:\path\to\pieces --original C:\path\to\full.png
```

无原图模式：

```powershell
python -m ctf_puzzle_solver --pieces C:\path\to\pieces
```

指定矩形网格：

```powershell
python -m ctf_puzzle_solver --pieces C:\path\to\pieces --rows 4 --cols 7
```

python -m ctf_puzzle_solver --pieces C:\path\to\pieces --no-auto

## 算法说明

有原图模式会先根据碎片数量、原图比例、碎片尺寸推断网格，或使用用户填写的行列数，再将每个碎片缩放采样后与原图网格区域进行均方误差评分，并用最小代价唯一分配算法选择整体最优放置。

无原图模式会提取每张碎片的上、下、左、右边缘像素，并计算颜色连续性和边缘变化差异。工具会用小宽度 beam search 构造候选矩形网格，再通过交换式局部优化降低整张图的总拼接代价。

乱拼整图拆分会根据纵横方向的边缘变化峰值寻找切割线。如果已知行列数，建议填写真实行列数，检测会在对应网格附近寻找最可能的碎片边缘。

边框对齐校准会优先使用自动拼接得到的行列位置，并按每一列、每一行的典型宽高累计生成网格线；没有拼接结果时，会根据当前碎片坐标吸附到最近的矩形网格位置。

该工具主要面向矩形切片、无旋转或少量需要手动旋转修正的 CTF 拼图题。复杂的不规则拼图仍建议结合手动拖动修正。

## 项目评估报告

### 当前完成度

Ez_Jigsaw v1.0 已实现 CTF 矩形拼图题的完整本地工作流：导入碎片、导入乱拼整图并自动拆分、可选完整原图辅助、无原图边缘拼接、手动拖动交换、边框校准、LSB bit-plane 查看、碎片导出、PNG 导出和进度条反馈。界面为深色工具台风格，顶部保留 `By Shr1mp` 作者标识。

### 算法能力

有原图模式的稳定性较高，因为每个碎片可以直接与原图网格区域做相似度匹配，再用唯一分配算法求整体最小误差。对于完整原图存在、碎片没有严重旋转或缩放失真的情况，该模式是推荐路径。

无原图模式已从简单边缘贪心扩展为多阶段算法：边缘颜色和梯度差异评分、置信度候选生成、横向/纵向条带候选、双向行对齐、整行偏移校准、局部行片段交换、低置信区域修复。该模式对局部纹理明显、边缘变化丰富的拼图有效；对大面积相似背景、重复头发/暗色块/纯色块较多的图片仍存在歧义。

乱拼整图拆分在规整网格上表现较好。实测 `C:\Users\q1388\Downloads\Test.jpg` 尺寸为 `4000 x 2400`，自动识别为 `12 x 20`，共 `240` 个 `200 x 200` 左右的矩形碎片。

### 分数说明

早期界面显示的“平均分”不是百分制，而是边缘差异代价，数值越低越好。为了避免误解，当前 UI 已改为显示 `质量分 x/100`。无原图模式的质量分由边缘代价换算得到：

```text
quality = 100 / (1 + edge_cost / 2500)
```

因此，`9500.95` 的边缘代价约等于 `20.8/100`。如果希望质量分接近 `98/100`，边缘代价需要降到约 `50` 左右；这通常需要完整原图辅助，或图片本身具有非常清晰且唯一的边缘连续性。

### 风险和局限

- 无原图模式无法保证全局最优，尤其是 `200+` 碎片的大图。
- 相似纹理、暗色区域、纯色背景和重复线条会让多个碎片边缘得分接近，导致局部正确但全局错位。
- 当前算法主要假设碎片为矩形、方向基本一致；大量旋转、翻转或不规则边缘需要更多人工修正。
- 质量分是基于边缘代价的估算，不等同于真实拼图正确率。
- 大图无原图拼接耗时较长，`240` 片级别测试约需 1 到 2 分钟，取决于机器性能。

### 后续优化方向

- 增加行列置换搜索，用于处理由整图按行列打乱得到的规则拼图。
- 引入 SSIM、互信息或局部特征点作为边缘评分补充，降低相似颜色区域的误配。
- 为低置信区域提供 UI 高亮，让用户快速定位需要手动修正的碎片。
- 增加多候选结果预览，让用户在若干自动拼接方案中选择最接近的一版。
- 对大图拼接加入可配置质量/速度模式，允许用户在速度和搜索深度之间取舍。
